    Informasi Status Batch
    Informasi Status Batch

    Berikut adalah penjelasan status SKU dan Order dalam sistem batch fulfillment:

    Status SKU
    SKU Selesai

    Total SKU yang sudah selesai diambil sesuai dengan jumlah yang dibutuhkan dalam batch.

    SKU Gantung

    SKU yang sudah diambil tetapi tidak dapat diproses karena masih kurang dari jumlah yang dibutuhkan untuk order.

    Over Stock

    SKU yang diambil melebihi jumlah yang dibutuhkan, biasanya terjadi karena order dibatalkan atau ada kesalahan dalam proses picking.

    Status Order
    Order to Pick

    Order yang sudah siap untuk diproses dan dapat dilakukan picking.

    Order Printed

    Order yang sudah siap untuk dipick dan telah dicetak (ready to print).

    Order Gantung

    Order yang belum siap untuk dipick karena masih menunggu persiapan atau ada masalah dengan stok.

    Kriteria Batch Fulfillment

    Batch dianggap sudah fulfill dan siap dibersihkan jika memenuhi semua kriteria berikut:

    ✓ SKU Selesai = Total SKU - Semua SKU sudah selesai diambil
    ✓ SKU Gantung = 0 - Tidak ada SKU yang tertunda
    ✓ Over Stock = 0 - Tidak ada kelebihan stok
    ✓ Order to Pick = Total Order - Semua order siap dipick
    ✓ Order Printed = Total Order - Semua order sudah dicetak
    ✓ Order Gantung = 0 - Tidak ada order yang tertunda
    Panduan Menghapus Order dari Batch

    Berikut adalah cara menghapus order dari batch berdasarkan status order:

    Case 1: Order Belum Printed (Status Order = pending)

    Kondisi: Order masih dalam status pending (belum dicetak)

    Lokasi: Halaman Batch Order (Orders)

    Cara: Klik tombol Hapus dari Batch di kolom Aksi

    Hasil: Order langsung dihapus dari batch, stok dikembalikan

    Case 2: Order Sudah Printed (Status Order = printed, Status tidak cancel)

    Kondisi: Order status printed dan status order tidak mengandung kata "cancel"/"batal"

    Lokasi: Halaman Scan Return Cancelled Order

    Cara: Gunakan form Scan Hapus Order Printed (warna kuning)

    Info: "Menghapus orderan yang sudah di printed saja dari Batch"

    Hasil: Order dihapus dari batch, stok dikembalikan, TIDAK update status return

    Case 3: Order Sudah Printed & Cancelled (Status Order = printed, Status = cancel/batal)

    Kondisi: Order status printed dan status order mengandung kata "cancel"/"batal"

    Lokasi: Halaman Scan Return Cancelled Order

    Cara: Gunakan form Scan Return Cancelled Order (warna biru)

    Hasil: Order dihapus dari batch, stok dikembalikan, DAN update status return

    Case 4: Order Sudah Diproses Lebih Lanjut

    Kondisi: Order status picked, packed, atau shipped

    Lokasi: Halaman Return List

    Cara: Wajib lewat Return Session di Return List

    Hasil: Proses return yang lebih kompleks dengan tracking lengkap

    Status Order Status Lokasi Action Keterangan
    pending Bebas Batch Order Hapus dari Batch Langsung hapus
    printed Tidak cancel Scan Return Cancelled Order Scan Hapus Order Printed Tidak update return status
    printed Cancel/Batal Scan Return Cancelled Order Scan Return Cancelled Order Update return status
    picked / packed / shipped Bebas Return List Return Session Wajib lewat return session
